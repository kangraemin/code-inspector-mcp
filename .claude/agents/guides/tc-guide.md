# TC 작성 및 검증 가이드

## TC 최소 기준 (Step당)
- 정상 케이스: 1개 이상
- 경계값 케이스: 1개 이상 (빈 배열, 0, null, 최대값 등)
- 에러/예외 케이스: 1개 이상 (잘못된 입력, 누락 필드 등)
- **최소 3개 TC** — 단순 Step이라도 예외 없음

## TC 테이블 필수 컬럼
| TC | 시나리오 | 입력값 | 기대 결과 | 검증 명령어 | 실제 출력 | 판정 |
- Step당 7컬럼 모두 채워야 함

## 검증 명령어 규칙
- 복붙해서 바로 실행 가능한 명령어여야 함
- `node -e "..."`, `python3 -c "..."`, `bash -c "..."` 등 인라인 실행 가능 형태
- 프로젝트 테스트 프레임워크 명령어도 가능 (pytest, jest 등)

## ✅ 찍기 전 체크리스트
1. 검증 명령어를 실제로 실행했는가?
2. 실제 출력을 step.md에 그대로 붙여넣었는가?
3. 기대 결과와 실제 출력이 일치하는가?
→ 3개 모두 YES일 때만 ✅. 하나라도 NO면 ❌.

## 경계값 예시
- 숫자: 0, 음수, 최대값, NaN, Infinity, 소수점
- 문자열: 빈 문자열, 공백만, 특수문자, 매우 긴 문자열, 유니코드
- 배열/리스트: [], [1개], 중복 요소, 매우 긴 배열
- 객체: {}, null, undefined, 필수 필드 누락, 추가 필드
- 파일: 존재하지 않는 경로, 빈 파일, 권한 없는 파일

## TC가 약한 패턴 (이런 TC는 반려됨)
- "정상 동작 확인" — 입력값/기대결과 없이 추상적 기술
- "에러 없이 실행됨" — 출력 내용 검증 없음
- "UI가 잘 보임" — 자동 검증 불가
- 동일 시나리오 변형만 3개 — 경계값/에러 없음
